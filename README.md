# 📚 QA Project: Testing Readeat Ecosystem
Даний репозиторій містить комплексну тестову документацію та приклади автоматизації для веб-платформи [Readeat.com](https://readeat.com/).

## 🎯 Мета проекту
Забезпечити високу якість користувацького досвіду (UX) та стабільність критичних бізнес-процесів: пошуку книг, роботи кошика та процесу оформлення замовлення (Checkout).

## 🛠 Технологічний стек
* **Manual Testing:** Складання Test Cases, Checklists, Bug Reports.
* **Automation:** Playwright (JavaScript).
* **Tools:** GitHub, Chrome DevTools, Postman.

---

## 📂 Зміст репозиторію

Нижче наведено посилання на основні артефакти тестування:

* [**📋 Тест-кейси (Test Cases)**](./test_cases.md) — детальні сценарії перевірки функціоналу (пошук, кошик, оплата).
* [**🐛 Звіти про помилки (Bug Reports)**](./bug_reports.md) — приклади оформлених дефектів, знайдених під час тестування.
* [**🔄 Регресійний чек-лист**](#-regression-checklist) — швидкий перелік перевірок перед релізом (знаходиться нижче).

---

## 🏁 Regression Checklist
Цей список використовується для швидкої перевірки стабільності основних функцій:
- [ ] Авторизація користувача (Email/Password).
- [ ] Додавання товару в кошик та коректний розрахунок суми.
- [ ] Робота пошуку за назвою книги.
- [ ] Вибір відділення Нової Пошти на сторінці Checkout.
- [ ] Перехід на платіжний шлюз після натискання "Оплатити".

---

## 🚀 Як запустити автотести (Playwright)
Якщо ви хочете переглянути приклад автоматизації, виконайте наступні команди:
1. Клонуйте репозиторій: `git clone https://github.com/ВАШЕ_ІМЯ/Readeat-QA-Project.git`
2. Встановіть залежності: `npm install`
3. Запустіть тести: `npx playwright test`
