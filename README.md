# 📚 QA Project: Testing Readeat Ecosystem
Даний репозиторій містить комплексну тестову документацію та приклади автоматизації для веб-платформи [Readeat.com](https://readeat.com/).

## 🎯 Мета проекту
Забезпечити високу якість користувацького досвіду (UX) та стабільність критичних бізнес-процесів: пошуку книг, роботи кошика та процесу оформлення замовлення (Checkout).

## 🛠 Технологічний стек
* **Manual Testing:** Складання Test Cases, Checklists, Bug Reports.
* **Automation:** Playwright (JavaScript).
* **Tools:** Google Sheets, GitHub, Chrome DevTools.
* **Methodology:** Functional, Regression, UI/UX Testing.

---

## 📂 Зміст репозиторію

### 1. Тестова документація
Всі кейси та перевірки структуровані за модулями:
* [**Test Cases Table**](./test-cases.md) — 15+ детальних сценаріїв (включаючи оплату частинами та передзамовлення).
* [**Regression Checklist**](./regression-checklist.md) — набір перевірок для швидкого тестування перед релізом.
* [**Bug Report Example**](./bug-reports/BUG-001.md) — приклад опису знайденого дефекту з пріоритезацією.

### 2. Автоматизація (Playwright)
У папці `/tests` знаходиться скрипт для автоматичної перевірки "Happy Path":
* Пошук книги за назвою.
* Додавання товару в кошик.
* Валідація переходу до оформлення.

> [!TIP]
> Щоб запустити автотести локально:
> 1. `npm install`
> 2. `npx playwright test`

## 📝 Результати тестування (Summary)
Під час роботи над проектом було проаналізовано:
